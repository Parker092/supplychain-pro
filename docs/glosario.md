# Glosario Técnico del Proyecto

## Arquitectura de sistemas

Disciplina que define la estructura general de un sistema de software, sus componentes, relaciones, responsabilidades, restricciones y decisiones técnicas principales.

## Backend

Componente encargado de procesar la lógica de negocio, recibir solicitudes, validar datos, comunicarse con la base de datos y exponer APIs para otros servicios.

## Base de datos

Sistema encargado de almacenar, organizar y proteger los datos de la aplicación.

## Cadena de frío

Conjunto de condiciones de temperatura controlada necesarias para conservar productos perecederos durante almacenamiento y transporte.

## CI

Integración continua. Práctica DevOps que permite validar automáticamente cambios en el código, por ejemplo construyendo imágenes Docker o ejecutando pruebas.

## Contenedor

Unidad de ejecución aislada que empaqueta una aplicación y sus dependencias para ejecutarse de forma consistente en distintos entornos.

## Docker

Plataforma que permite crear, ejecutar y administrar contenedores.

## Docker Compose

Herramienta que permite definir y ejecutar múltiples contenedores usando un archivo de configuración.

## Frontend

Componente visual de la aplicación con el que interactúa el usuario.

## Geofencing

Técnica que permite validar si una ubicación GPS se encuentra dentro o fuera de un perímetro definido.

## GitHub Actions

Plataforma de automatización integrada en GitHub que permite ejecutar flujos de CI/CD.

## Incidente de calidad

Registro generado cuando el sistema detecta una condición que puede afectar la calidad del producto transportado.

## Inmutabilidad

Propiedad de un registro que impide su modificación o eliminación después de haber sido creado.

## Node.js

Entorno de ejecución de JavaScript usado para crear servicios backend y scripts de automatización.

## PostgreSQL

Sistema de gestión de bases de datos relacional utilizado para almacenar información persistente del proyecto.

## Red interna

Red privada creada por Docker para permitir comunicación entre contenedores sin exponer todos los servicios al exterior.

## Resiliencia

Capacidad de un sistema para recuperarse o continuar operando después de un fallo.

## Simulador

Componente que genera datos ficticios de telemetría para representar el comportamiento de un camión virtual.

## Telemetría

Datos generados por un dispositivo o sistema remoto, como temperatura, humedad, batería y coordenadas GPS.

## Trigger

Mecanismo de base de datos que ejecuta una acción automáticamente cuando ocurre un evento, como una actualización o eliminación.

## Volumen Docker

Mecanismo de almacenamiento persistente utilizado para conservar datos fuera del ciclo de vida de un contenedor.
